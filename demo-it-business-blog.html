Questions fréquentes

Qu’est-ce que le modèle HaaS ?

HaaS (« Hawaii as a Service ») combine un forfait projet pour la conception & mise en production de votre application, puis un abonnement Run couvrant l’hébergement Azure, la supervision, le support et les petites évolutions. Vous concentrez vos équipes sur le métier, nous gérons la technique.

Quelle différence entre les formules Basic et Premium ?

Basic : ticketing Jira, suivi des coûts, 1 accès licence M365, frais fixes réduits.

Premium : SLA, interlocuteur dédié, 2 accès support, options de paiement élargies, carnet d’heures dev & support applicatif.

Est-ce que la consommation Azure est incluse dans le prix ?

La consommation est refacturée au coût réel (Pay-As-You-Go). Nos frais de gestion (10 % conso) couvrent la surveillance FinOps, la facturation simplifiée et l’optimisation continue.

Quels délais pour un projet type ?

Analyse métier : 1–2 semaines

Conception UX/UI : 2–4 semaines

Développement Agile : 2–4 sprints (4–8 semaines) selon périmètre

Mise en prod : 1 semaine

→ Une V1 est donc livrable en 8–12 semaines.

Qui possède le code source ?

Vous. Le dépôt Git (Azure DevOps ou GitHub) vous appartient. Nous fournissons toute la documentation pour assurer la réversibilité.

Comment gérez-vous la sécurité ?

Sécurité « by design » : DevSecOps, scan dépendances, secrets dans Key Vault, RBAC Entra ID, Defender & Sentinel activés. Les données restent dans votre tenant Azure.

Et si notre trafic explose ?

Les services PaaS (Web Apps, Functions, SQL) scalent automatiquement. Vous payez uniquement la consommation supplémentaire. Nous ajustons les quotas si nécessaire.

Avez-vous une démarche FinOps / Green IT ?

Oui : droit-sizing automatique, environnements hors prod mis en veille, rapports mensuels CO₂ & coût. Objectif : – 20 % d’empreinte carbone la 1ʳᵉ année.

Pouvez-vous intégrer des outils non-Microsoft ?

Bien sur. Via APIs REST, connecteurs custom Power Platform ou micro-services, nous interfaçons Salesforce, SAP, AWS S3, ou n’importe quelle solution SaaS disposant d’une api..

Comment se passe le support au quotidien ?

Création de ticket dans Jira Service Desk Priorisation : P1 (< 2h), P2 (< 8h), P3 (J+1)

Reporting mensuel sur SLA, coûts et roadmap évolutive.

Proposez-vous des formations ?

Oui : sessions Power BI, Power Apps, FinOps Azure, Sécurité Azure, prises en charge OPCO possibles.

Checker formations BC (+Green IT)

Quelle est votre zone d’intervention ?

Nous opérons à distance partout en France (et Europe). Présence physique possible à Paris et déplacements ponctuels sur site.

Comment démarrer ?

Un simple rendez-vous (30 min) pour qualifier vos enjeux, puis un devis forfaitaire ou un sprint 0 : analyse + backlog + macro-planning.
